================================================================================
USAGE INSTRUCTIONS
================================================================================

1. INSTALL DEPENDENCIES:
   cd D:\Projects\obs-dndbeyond-automation
   npm install

2. CONFIGURE ENVIRONMENT:
   cp .env.example .env
   [Edit .env with your credentials]

3. START APPLICATION:
   npm start              # Production mode
   npm run dev            # Development with auto-reload
   npm run build          # Compile TypeScript

4. REQUIRED CONFIGURATION:
   - DND_CHARACTER_ID: From D&D Beyond character URL
   - DND_COBALT_SESSION: From browser DevTools → Network → character-service request
   - OBS_MODE: "image_swap" or "visibility_toggle"
   - OBS mode-specific variables (see .env.example)

5. OBS SETUP:
   - Enable WebSocket Server: Tools → WebSocket Server Settings → Enable
   - For visibility_toggle: Create scene items named: healthy, scratched, 
     bloodied, dying, unconscious
   - For image_swap: Ensure image source exists with specified name

================================================================================
ARCHITECTURE DECISIONS
================================================================================

1. ASYNC/AWAIT THROUGHOUT
   - All network operations are async
   - No callback hell or promise chains
   - Cleaner error handling with try/catch
   - Better code readability

2. SEPARATE CONCERNS
   - config.ts: Configuration validation
   - types.ts: Type definitions
   - dnd-beyond/: D&D Beyond specific logic
   - obs/: OBS specific logic
   - index.ts: Application orchestration
