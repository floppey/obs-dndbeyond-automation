Overview

OBS D&D Beyond Automation is a tool that monitors your D&D Beyond character sheet in real-time and automatically updates your OBS scenes based on rules you define. It bridges the gap between your digital character data and your streaming setup.

How it works:
  • The app polls your D&D Beyond character every few seconds
  • Character data (HP, equipment, status) is compared against your rules
  • When rule conditions match, corresponding OBS actions are triggered
  • OBS updates are instant, providing immersive visual feedback to your stream

Key Features:

  • 📊 Character State: Live view of your character's HP, equipment, and status
  • ⚙️ Rules: Visual editor for creating automation rules
  • ⚡ Settings: Configuration for D&D Beyond and OBS connections
  • ❓ Help: This comprehensive documentation

D&D Beyond Setup

To connect to your D&D Beyond character, you need two pieces of information:

1. Character ID

Your unique character identifier from D&D Beyond.

How to find it:
  1. Go to your D&D Beyond character sheet (dndbeyond.com)
  2. Look at the URL in your browser's address bar
  3. The Character ID is the number at the end of the URL
  4. Example: dndbeyond.com/characters/139364185/Oota → ID is 139364185

2. Cobalt Session Cookie

An authentication token that allows the app to access your character data privately.

How to get it:
  1. Open your browser's Developer Tools (F12 or Right-click → Inspect)
  2. Go to the Application tab (or Storage in Firefox)
  3. In the left sidebar, expand Cookies
  4. Find dndbeyond.com and click it
  5. Look for the cobaltSession cookie
  6. Copy the entire value (it's a long string)
  7. Paste it into the Cobalt Session field in Settings
⚠️ Important: Your cobalt session cookie acts like a password. Keep it secret! The app shows only the last 10 characters when saved to help you verify it's correct.
⚠️ Session Expiry: Your cobalt session may expire periodically. If the app stops receiving character data, you'll need to get a new cookie and update Settings.

OBS Setup

Your OBS instance must have the WebSocket Server enabled for the app to communicate with it.

1. Enable WebSocket Server

  1. Open OBS
  2. Go to Tools → WebSocket Server Settings
  3. Check the box: Enable WebSocket Server
  4. Note the Server Port (default: 4455)
  5. Optionally set a password for security
  6. Click Apply or OK

2. Configure in Settings

In the app's ⚡ Settings tab, configure:

  • WebSocket URL: Should be ws://localhost:4455 for local OBS
  • WebSocket Password: If you set one in OBS settings (leave empty if not)
  • Scene Name: The OBS scene containing your character display
  • Source Name: The specific OBS source/item to update

3. Scene and Source Naming

OBS organized content using Scenes and Sources. Understanding this structure is key:

  • Scene: A collection of sources (e.g., "Combat", "Roleplay", "Dungeon")
  • Source: Individual elements like images, text, groups (e.g., "Character_Portrait", "HP_Bar")
  • The app updates specific sources within specific scenes
  • Source names must match exactly (case-sensitive)
To find source names in OBS:
  1. Navigate to the scene containing your target source
  2. Look in the Sources dock (usually bottom-left)
  3. Source names appear in the list exactly as configured

Rules Engine

Rules are the core of automation. They define "when something happens" (conditions) and "what to do about it" (actions).

Rule Anatomy

{
  "name": "Show Bloodied Portrait",
  "enabled": true,
  "priority": 50,
  "condition": { "type": "hp_percentage", "operator": "<=", "value": 50 },
  "actions": [
    { "type": "set_image", "sourceName": "Portrait", "imagePath": "images/bloodied.png" }
  ]
}

Rule Lists and Modes

Rule Lists are groups of related rules that share a processing mode:

  • first_match: Stops at the first rule that matches (useful for mutual exclusions)
  • all_matches: Runs all matching rules (useful for stacking effects)

Example: A portrait rule list with mode=first_match ensures only one portrait displays at a time.

Priority

Within a rule list, rules are evaluated in order of priority:

  • Higher numbers = evaluated first (0-100)
  • Default priority is 0
  • Use priority to ensure more specific conditions are checked first

Example: Check "is_dead" (priority 100) before "hp_percentage" (priority 50)

Conditions

Conditions check the character state. All must be true for a rule to execute (AND logic):

  • Single conditions: Simple checks (HP < 50%, has armor, etc.)
  • AND groups: All conditions must match
  • OR groups: Any condition can match
  • Complex combinations: Mix AND/OR groups for complex logic

Example: "If HP ≤ 50% AND character is not dead OR temporary HP > 0"

Actions

Actions are executed when all conditions match:

  • Each rule can have multiple actions
  • Actions execute in order
  • Actions are only executed if conditions match
Rule Editor Tips:
  • Use the visual editor for simple rules
  • Use Advanced JSON editor for complex conditions
  • Test rules by changing your character state in D&D Beyond
  • Check OBS to confirm sources update correctly

Condition Types Reference

All 16 condition types with examples of when to use them:

Type Description Parameters Example
hp_percentage HP as % of max operator, value (0-100) <= 50 → Bloodied
hp_value Current HP amount operator, value < 10 → Critical
hp_temp Temporary HP operator, value > 0 → Has temp HP
hp_missing Damage taken (max - current) operator, value > 20 → Significant damage
is_dead Character is dead value (true/false) true → Show death overlay
is_unconscious Character unconscious value (true/false) true → Show unconscious portrait
death_saves_success Death save successes operator, value (0-3) == 3 → Stable
death_saves_failure Death save failures operator, value (0-3) == 3 → Character dies
item_equipped Item is equipped itemName, matchPartial Sword, partial=false → Has sword
item_attuned Item is attuned itemName, matchPartial Ring, partial=true → Any ring attuned
armor_equipped Any armor equipped value (true/false) true → In armor
shield_equipped Shield equipped value (true/false) true → Has shield
level Total character level operator, value >= 10 → Double digits!
class_level Specific class level className, operator, value Wizard >= 3 → 3rd level spells
has_class Has specific class className, value (true/false) Barbarian, true → Is Barbarian
always Always true (default) (none) No condition = always run

Operators for Comparison Conditions

For conditions with numeric values, use these operators:

  • > Greater than
  • >= Greater or equal
  • < Less than
  • <= Less or equal
  • == Exactly equal
  • != Not equal
Item Matching:
  • matchPartial = false: Item name must match exactly
  • matchPartial = true: Item name can be partial (searches for substring)
  • Example: "Ring" with partial=true matches "Ring of Protection", "Magical Ring", etc.

Action Types Reference

Four types of actions that modify your OBS scene:

Type Description Parameters Example
set_image Change image source sourceName, imagePath Portrait → images/bloodied.png
set_visibility Show/hide scene item sceneName, itemName, visible Scene: Combat, Item: Dead_Overlay, visible: true
set_text Set text content sourceName, text (supports variables) HP_Display → "HP: {currentHp}/{maxHp}"
set_filter_visibility Enable/disable filter sourceName, filterName, visible Portrait + Red Tint filter → visible: true

Text Variables

In set_text actions, you can use these variables in the text:

  • {currentHp} Current HP value
  • {maxHp} Maximum HP
  • {hpPercentage} HP as percentage
  • {tempHp} Temporary HP
  • {isDead} true/false
  • {isUnconscious} true/false
  • {level} Total level
  • {className} Primary class name

Example: "HP: {currentHp}/{maxHp} ({hpPercentage}%)" → "HP: 28/50 (56%)"

Action Tips:
  • Image paths can be local files (e.g., E:/OBS/portrait.png) or URLs
  • Source and scene names are case-sensitive
  • Multiple actions execute in order
  • Test each action individually before combining

Example Use Cases

Common automation scenarios and how to set them up:

Example 1: HP-Based Portrait Switching

Show different character portraits based on HP status (healthy → bloodied → dying → unconscious).

Setup:

  • Create a Rule List: "Portrait States"
  • Mode: first_match (only one portrait at a time)
{
  "name": "Portrait States",
  "enabled": true,
  "mode": "first_match",
  "rules": [
    {
      "name": "Dead Portrait",
      "priority": 100,
      "condition": { "type": "is_dead", "value": true },
      "actions": [{ "type": "set_image", "sourceName": "Portrait", "imagePath": "images/dead.png" }]
    },
    {
      "name": "Unconscious Portrait",
      "priority": 90,
      "condition": { "type": "is_unconscious", "value": true },
      "actions": [{ "type": "set_image", "sourceName": "Portrait", "imagePath": "images/unconscious.png" }]
    },
    {
      "name": "Dying Portrait",
      "priority": 80,
      "condition": { "type": "hp_percentage", "operator": "<=", "value": 25 },
      "actions": [{ "type": "set_image", "sourceName": "Portrait", "imagePath": "images/dying.png" }]
    },
    {
      "name": "Bloodied Portrait",
      "priority": 70,
      "condition": { "type": "hp_percentage", "operator": "<=", "value": 50 },
      "actions": [{ "type": "set_image", "sourceName": "Portrait", "imagePath": "images/bloodied.png" }]
    },
    {
      "name": "Healthy Portrait",
      "priority": 60,
      "condition": { "type": "always" },
      "actions": [{ "type": "set_image", "sourceName": "Portrait", "imagePath": "images/healthy.png" }]
    }
  ]
}

Example 2: Equipment-Based Effects (Polymorph)

Change character appearance based on equipped items (e.g., transforming via polymorph).

{
  "name": "Equipment Effects",
  "enabled": true,
  "mode": "first_match",
  "rules": [
    {
      "name": "Crocodile Form",
      "priority": 100,
      "condition": { "type": "item_equipped", "itemName": "Polymorph Amulet", "matchPartial": true },
      "actions": [
        { "type": "set_image", "sourceName": "Portrait", "imagePath": "images/croc.png" },
        { "type": "set_text", "sourceName": "StatusDisplay", "text": "CROCODILE FORM" }
      ]
    },
    {
      "name": "Normal Form",
      "priority": 0,
      "condition": { "type": "always" },
      "actions": [
        { "type": "set_image", "sourceName": "Portrait", "imagePath": "images/normal.png" },
        { "type": "set_text", "sourceName": "StatusDisplay", "text": "Normal" }
      ]
    }
  ]
}

Example 3: Death Indicator Overlay

Show/hide a death overlay when character dies.

{
  "name": "Death Overlay",
  "enabled": true,
  "mode": "all_matches",
  "rules": [
    {
      "name": "Character Dead",
      "priority": 100,
      "condition": { "type": "is_dead", "value": true },
      "actions": [
        { "type": "set_visibility", "sceneName": "Combat", "itemName": "DeathOverlay", "visible": true }
      ]
    },
    {
      "name": "Character Alive",
      "priority": 100,
      "condition": { "type": "is_dead", "value": false },
      "actions": [
        { "type": "set_visibility", "sceneName": "Combat", "itemName": "DeathOverlay", "visible": false }
      ]
    }
  ]
}

Example 4: Live HP Display

Update a text source with current HP and percentage.

{
  "name": "HP Display",
  "enabled": true,
  "mode": "all_matches",
  "rules": [
    {
      "name": "Update HP Text",
      "priority": 0,
      "condition": { "type": "always" },
      "actions": [
        { "type": "set_text", "sourceName": "HP_Text", "text": "HP: {currentHp}/{maxHp} ({hpPercentage}%)" }
      ]
    }
  ]
}

Troubleshooting

Solutions to common issues:

Connection Lost / App Won't Connect

Problem: Status shows "Disconnected" or red dot

Solutions:

  • Check that the app is actually running (command line window open)
  • Verify OBS is open and WebSocket Server is enabled
  • Check the WebSocket URL in Settings matches your OBS setup
  • If using a password, verify it's correct in Settings
  • Try restarting both the app and OBS
  • Check your firewall isn't blocking port 4455

Character Data Not Loading

Problem: Character State shows "Waiting for character data..."

Solutions:

  • Verify Character ID is correct (check in D&D Beyond URL)
  • Get a fresh Cobalt Session cookie (old ones expire)
  • Make sure the character sheet is public or you're logged into D&D Beyond
  • Try changing something on your character sheet in D&D Beyond (take damage, equip item)
  • Check network connection - the app needs internet access

Rules Not Triggering

Problem: OBS sources don't update when conditions should match

Solutions:

  • Verify the rule is enabled (toggle should be ON)
  • Check the rule list is enabled
  • Verify the condition matches your current character state
  • Check condition values (e.g., is HP actually <= 50%?)
  • Test the condition in Character State tab to see live values
  • Try disabling and re-enabling the rule
  • Temporarily change your character in D&D Beyond to trigger the condition

OBS Sources Not Updating

Problem: Rules trigger but OBS doesn't change

Solutions:

  • Verify OBS WebSocket connection is active (green dot in app status)
  • Check source names are spelled exactly (case-sensitive)
  • Verify source exists in OBS at the location you specified
  • For images, confirm the file path is correct (try full absolute path)
  • Try updating the source manually in OBS to confirm it works
  • Check OBS logs for WebSocket errors (Help → Log Files)
  • Restart OBS and reconnect the app

Performance Issues

Problem: App is slow or unresponsive

Solutions:

  • Increase polling interval in Settings (default 5000ms is fine)
  • Don't use complex OR groups with many conditions
  • Keep rule lists focused - don't mix unrelated rules
  • Close browser developer tools (they slow things down)
  • Check system resource usage (CPU, memory)

Getting More Help

Still stuck?

  • Check Character State tab to see live character data
  • Export your rules (Rules tab) and review the JSON
  • Enable "Save API Responses" in Settings Debug section
  • Check the command line window for error messages
  • Review OBS logs and browser console (F12) for errors
